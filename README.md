# 🚗 FuturePark - AI-Powered Smart Parking System

This repository contains the project management documentation and planning artifacts for **FuturePark**, an AI-powered smart parking system developed for the Higher Colleges of Technology (HCT). This project was completed as part of the "Project Management (CIS 3603)" course.

---

## 🚀 Project Overview

The FuturePark project aims to develop a smart parking system to alleviate the daily stress and wasted time of finding a parking space for HCT students, staff, and faculty. The core deliverable is a mobile application that provides real-time parking guidance and advance reservations, supported by License Plate Recognition (LPR) cameras and IoT sensors.

While the overall project follows a hybrid methodology (Waterfall for hardware, Agile for software), this repository focuses on the **Agile (Scrum)** framework used for the software development components.

### Key Objectives Achieved:
- **Reduced Search Time:** Successfully implemented a real-time parking system using sensors to guide users directly to available spots, aiming for a search time of under 5 minutes.
- **Congestion Reduction:** Partially achieved the goal of reducing morning peak congestion by 40%, with full validation pending a complete semester of usage.

---

## 🛠️ Project Management Framework & Methodology

We adopted an **Agile (Scrum)** approach to manage the software development lifecycle, allowing for continuous adaptation based on user feedback.

- **Iterative Planning:** Work was divided into three 2-week Sprints.
- **Incremental Delivery:** Working features were delivered at the end of each Sprint.
- **Adaptive Control:** The Product Backlog was continuously refined based on Sprint Reviews.
- **Cadence:** Regular ceremonies included Sprint Planning, Daily Scrums, Sprint Reviews, and Sprint Retrospectives.

---

## 📊 Scope & Work Breakdown Structure (WBS)

The project scope was managed dynamically through a prioritized Product Backlog. High-priority User Stories were broken down into actionable tasks for each Sprint.

### Sprint Highlights:
*   **Sprint 1 (Core Functionality):** Focused on user registration, real-time parking map, and automatic gate access via LPR integration.
*   **Sprint 2 (Reservation & Admin):** Implemented parking reservations, in-app navigation, and the administrative dashboard.
*   **Sprint 3 (Finalization):** Focused on completing remaining tasks, improving performance, and preparing for final delivery.

## ⚠️ Risk Management

A comprehensive Risk Register was maintained. Key risks identified included:
1.  **Changing Requirements:** Mitigated through regular backlog refinement and stakeholder involvement.
2.  **Integration Issues (LPR & IoT):** Mitigated via early integration and testing.
3.  **Resource Availability:** Managed through proper task planning and workload distribution.

## 💡 My Role & Key Learnings

As the **Project Manager**, my primary responsibilities included:
*   Leading the project, managing the overall schedule and budget.
*   Serving as the main communication link between the team and external stakeholders.
*   Organizing work for deliverables, setting clear deadlines, and ensuring team alignment.
*   Monitoring project performance using MS Project and EVA metrics.

### Key Lessons Learned:
1.  **Detailed Task Estimation:** Breaking down large features into smaller technical tasks and adding a time buffer significantly improved estimation accuracy (moving from optimistic estimates to realistic ones using PERT).
2.  **The Power of Daily Communication:** Enforcing strict Daily Scrum meetings drastically reduced blockers and improved team collaboration.
3.  **Value of Stakeholder Feedback:** Involving end-users in Sprint Reviews prevented the development of confusing UI features and ensured the final product was user-friendly.
